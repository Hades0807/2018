What is Shiny?

  • Shiny is a web application framework for R.
  • Shiny allows you to create a graphical interface so that users can interact with your visualizations, models, and algorithms without needed to know R themselves.
  • Using Shiny, the time to create simple, yet powerful, web-based interactive data products in R is minimized.
  • Shiny is made by the fine folks at R Studio.

Some Mild Prerequisites

Shiny doesn't really require it, but as with all web programming, a little knowledge of HTML, CSS and Javascript is very helpful.

  • HTML gives a web page structure and sectioning as well as markup instructions
  • CSS gives the style
  • Javscript for interactivity

Shiny uses Bootstrap (no relation to the statistics bootstrap) style, which (to me) seems to look nice and renders well on mobile platforms.

A Shiny project

A shiny project (before version 0.10.2) is a directory containing at least two files:

  • ui.R (for user interface) controls how your app looks.
  • server.R that controls what your app does.

The newest version of shiny project is a directory containing one file app.R, where two objects are defined:

  • ui (for user interface) controls how your app looks.
  • server that controls what your app does.

We focus on introducing the newest version.

App with Inputs and Outputs

Now that you've played around with customizing the look of your app, let's give it some functionality! Shiny provides several types of inputs including buttons, checkboxes, text boxes, and calendars. First let's experiment with the slider input. This simple app will display the number that is selected with a slider.

Slider App

Slider App: ui

library(shiny)
ui <- fluidPage(
  titlePanel("Slider App"),
  sidebarLayout(
    sidebarPanel(
      h1("Move the Slider!"),
      sliderInput("slider1", "Slide Me!", min = 0, max = 100, value = 0)
    ),
    mainPanel(
      h3("Slider Value:"),
      textOutput("text")
    )
  )
)

Slider App: server

library(shiny)
server <- function(input, output) {
  output$text <- renderText(input$slider1)
}

New Components in the Slider App

ui

  • sliderInput() specifies a slider that a user can manipulate
  • textOutput() displays text that is rendered in server.R

server

  • renderText() transforms UI input into text that can be displayed.

Inputs and Outputs

Notice that in ui input and output names are assigned with strings (sliderInput("slider1",..., textOutput("text")) and in server those components can be accessed with the $ operator: output$text <- renderText(input$slider1).

Apps with Plots

Allowing users to manipulate data and to see the results of their manipulations as a plot can be very useful. Shiny provides the plotOutput() function for ui and the renderPlot() function for sever for taking user input and creating plots.

Plot App: server Part 1

library(shiny)
server <- function(input, output) {
  output$plot1 <- renderPlot({
    set.seed(2018-12-10)
    number_of_points <- input$numeric
    minX <- input$sliderX[1]
    maxX <- input$sliderX[2]
    minY <- input$sliderY[1]
    maxY <- input$sliderY[2]
# ...

Plot App: server Part 2

# ...
    dataX <- runif(number_of_points, minX, maxX)
    dataY <- runif(number_of_points, minY, maxY)
    xlab <- ifelse(input$show_xlab, "X Axis", "")
    ylab <- ifelse(input$show_ylab, "Y Axis", "")
    main <- ifelse(input$show_title, "Title", "")
    plot(dataX, dataY, xlab = xlab, ylab = ylab, main = main,
         xlim = c(-100, 100), ylim = c(-100, 100))
  })
}
